◆ 공지사항 게시판(notice_board)
- 검색창
- 페이징
- 게시물 개수 페이지별로 10개씩만 띄우기
- 글쓰기는 관리자로그인시에만 보이게 만들기 (M_MANAGERCHK='Y')
- viewcount 만들기
- viewcount 볼때마다 업데이트되게 업데이트 문 만들기

◆ 공지사항 글(notice_detail)
- 삭제버튼 설정
- 수정되었으면 등록일에 [(수정)날짜] 될 수 있게 만들기
- 삭제 버튼 누르면 데이터 삭제 되게 만들기
- 수정버튼은 본인의 게시물만 수정할수 있게 띄우기
- 작성자 id값으로 받았기때문에 이름으로 표기 변경해주기
- 도서관 id값으로 받았기때문에 이름으로 표기 변경해주기

◆ 공지사항 글쓰기(notice_write)
- 첨부파일 넣으면 삭제도 할 수 있게 만들기
- 첨부파일 한개만 넣는게아니라 여러개 넣을 수 있게 만들기
- 로그인 했을때의 관리자 이름 가져오기
- 유효성 검사 자바스크립트로 하기 form버리기

◆ 공지사항 수정(notice_update)
- 수정 완료되면 데이터 수정되기
- 첨부파일 한개만 넣는게아니라 여러개 넣을 수 있게 만들기
- 첨부파일 데이터 어떻게 끌고올것?

◆ QnA 게시판(QnA_board)
- 검색창
- 페이징
- 게시물 개수 페이지별로 10개씩만 띄우기
- viewcount 만들기
- viewcount 볼때마다 업데이트되게 업데이트 문 만들기

◆ QnA 글(QnA_detail)
- 삭제버튼 설정
- 수정되었으면 등록일에 [(수정)날짜] 될 수 있게 만들기
- 답글 버튼 누른 후 등록 되게 만들기
- 답글의 삭제버튼은 답글 달땐 안보이게 하기
- 답글 등록되면 삭제버튼 뜨게 만들기
- 작성자 id값으로 받았기때문에 이름으로 표기 변경해주기
- 도서관 id값으로 받았기때문에 이름으로 표기 변경해주기
- 수정 버튼 누르면 비밀글은 비밀번호 입력하고 수정할 수 있게 만들기/공개글은 그냥 만들기

◆ QnA 글쓰기(QnA_write)
- 첨부파일 넣으면 삭제도 할 수 있게 만들기
- 첨부파일 한개만 넣는게아니라 여러개 넣을 수 있게 만들기

◆ QnA 수정(QnA_update)
- 첨부파일 한개만 넣는게아니라 여러개 넣을 수 있게 만들기
- 첨부파일 데이터 어떻게 끌고올것?

◆ 희망도서 신청(wishbook_add)
- 이름, 번호 데이터 가져오기
- 로그인되었을때만 희망도서 신청 가능하게 만들기
- 로그인 안되어있으면 로그인페이지로 이동시키기
